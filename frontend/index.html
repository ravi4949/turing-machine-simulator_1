Core concepts behind the Turing Machine model

📼

The Tape

An infinite sequence of cells, each storing one symbol from the tape alphabet Γ. The tape is initially populated with the input string; all other cells hold the blank symbol (B).

🔵

States (Q)

A finite set of internal configurations the machine can be in. Always includes a designated start state q₀, an accept state q_a, and a reject state q_r.

δ

Transition Function δ

The "program" of the machine: δ: Q × Γ → Q × Γ × {L, R}. Given the current state and tape symbol, it produces a new state, writes a symbol, and moves the head Left or Right.

✓

Acceptance

A string w is accepted if the machine enters q_accept at any point during computation. It is rejected if it enters q_reject or if no transition is defined (implicit reject).

⚙️

Configuration

A snapshot of the machine at any moment: (current state, tape contents, head position). The simulation history is a sequence of configurations, one per step.

∞

Turing-Completeness

A system is Turing-complete if it can simulate any Turing Machine. Modern CPUs, programming languages, and even cellular automata (Rule 110) are all Turing-complete.

Formal 7-Tuple Definition

M = (Q, Σ, Γ, δ, q₀, q_accept, q_reject)

  • Q — finite set of states
  • Σ — input alphabet (does not contain blank B)
  • Γ — tape alphabet (Σ ⊂ Γ, B ∈ Γ)
  • δ — transition function: Q × Γ → Q × Γ × {L, R}
  • q₀ — start state (q₀ ∈ Q)
  • q_accept — accept state
  • q_reject — reject state (q_reject ≠ q_accept)
